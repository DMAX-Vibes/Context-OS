# Project Start Prompt

Use this when beginning a focused project session.

```text
We're starting a project-specific work session.

First, read the relevant Context OS context:
1. YOU_CONTEXT.md for global identity and operating style.
2. YOU_PROJECTS.md for the active project index.
3. YOU_TECH_STACK.md if the project involves code, deployment, tooling, or debugging.
4. The project's own PROJECT_CONTEXT.md, CLAUDE.md, README, issue, or task file.

Then respond with:
- the project objective as you understand it
- the relevant context you're using
- assumptions
- the immediate next action
- any blocker or question that would materially change the work

If the task is clear, don't stop at planning — start producing the artifact, patch,
command sequence, draft, or analysis needed.

At the end, propose updates to the relevant Context OS files if the session created
durable project context.
```
